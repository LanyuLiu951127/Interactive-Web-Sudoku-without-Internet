# 需求文件 v2 (REQ_v2.md) - 純前端架構版

## 1. 專案概述
本專案已升級為「全靜態/純前端」架構，核心數獨邏輯（生成、求解、驗證）完全由瀏覽器端的 JavaScript 執行。這項改變旨在達成 100% 的離線支援，並能無縫部署於任何靜態網頁代管空間（如 `st.sweb.name`）。

## 2. 功能需求 (Functional Requirements)
*   **F01: 數獨生成 (本機化)**: 系統必須能自動生成具有唯一解的 9x9 數獨題目，且不依賴伺服器。
*   **F02: 難度選擇**: 提供「簡單」、「中等」、「困難」三種難度。
*   **F03: 互動棋盤**: 玩家可以點擊單元格並透過虛擬鍵盤或實體鍵盤輸入數字 (1-9)。
*   **F04: 視覺提示**: 當玩家輸入與答案不符或違反數獨規則時，給予即時視覺提示。
*   **F05: 極致離線支援**: 支援在完全無網路環境下開啟並遊玩，所有邏輯皆在本地執行。
*   **F06: 筆記模式**: 玩家可以在單元格中標註可能的數字。
*   **F07: 遊戲計時**: 記錄玩家完成遊戲所花費的時間。
*   **F08: 本地進度保存**: 利用 LocalStorage 自動保存當前遊戲進度，無需帳號系統。
*   **F09: 獲勝判斷**: 當棋盤填滿且完全正確時，顯示獲勝訊息。
*   **F10: 無限撤銷 (Undo)**: 提供返回上一步功能。
*   **F11: 快速清除 (Erase)**: 清除選定格子的內容。
*   **F12: 互動式提示 (Hint)**: 透過多步驟引導，幫助玩家學習解題邏輯而非直接給答案。
*   **F13: 錯誤計數器**: 記錄填錯次數。
*   **F14: 遮罩暫停**: 暫停期間隱藏盤面。
*   **F15: 求解器 (Client-side Solver)**: 允許玩家輸入外部題目，並由前端演算法即時運算解答，無需透過後端 API。

## 3. 非功能需求 (Non-functional Requirements)
*   **NF01: 跨平台響應式**: 適配各種螢幕尺寸。
*   **NF02: 零伺服器負載**: 所有運算均在客戶端完成，節省伺服器成本。
*   **NF03: 瞬間啟動**: 由於無需等待網路回傳題目，新遊戲生成速度極快。
*   **NF04: 資料隱私**: 玩家數據僅存於本地瀏覽器，不會上傳至伺服器。
