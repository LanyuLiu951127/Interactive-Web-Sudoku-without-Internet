# 需求規格說明書 (REQ.md)

## 1. 專案概述 (Project Overview)
本專案為一個全端互動式數獨 Web 應用程式，專為桌上型電腦及移動裝置（如 iPhone 12 Pro）進行極致的 UI/UX 優化。系統具備無網路狀態下的強健單機遊玩能力，同時提供基於安全資料庫（PHP 8.4 + MariaDB 10.11）的全端使用者管理、好友通訊，與內建防作弊異常監控的系統日誌機制。本專案以極致美學（Glassmorphism 磨砂玻璃、動態 HSL 漸層角色徽章、微型 RWD 滾動條）打造頂級的使用者體驗。

---

## 2. 核心功能需求 (Functional Requirements)

### 2.1 遊戲核心與輔助功能
*   **F01: 本端數獨生成**: 核心生成邏輯完全移至客戶端 JavaScript (Backtracking 演算法)，不依賴伺服器請求，確保零延遲並具備唯一解。
*   **F02: 三段難度選擇**: 提供「簡單（40-45 提示數）」、「中等（30-35 提示數）」與「困難（22-28 提示數）」三種挑戰難度。
*   **F03: 互動式棋盤**: 支援觸控（虛擬九宮格數字鍵）與實體鍵盤輸入，點選單元格時，關聯行、列、宮格會呈現柔和的高亮，且相同數字會一併高亮。
*   **F04: 步驟復原與擦除 (Undo & Erase)**: 提供無限制次數的步驟撤銷（包含筆記與數字填入歷史棧），並可單點擦除特定單元格。
*   **F05: 智慧型筆記模式 (Notes Mode)**: 允許玩家在空白格子中以草稿標記多個候選數。當填入正式數字時，關聯格子的對應筆記會自動擦除。
*   **F06: 漸進式互動教學 (Guided Hint)**: 當卡關時提供引導式提示，不直接給答案，而是透過四個視覺化步驟（目標定位、關聯高亮、衝突排除、邏輯推導）教育玩家解題邏輯。
*   **F07: 進度自動保存 (Auto-save & Resume)**: 系統每 5 秒及每次輸入時自動保存棋盤狀態與計時器至 `LocalStorage`，重開網頁無縫恢復。
*   **F08: 遮罩暫停機制**: 玩家可暫停計時器，暫停期間棋盤會被毛玻璃遮蔽，防止透過截圖作弊。

### 2.2 數獨智慧求解器 (Sudoku Solver)
*   **F09: 即時衝突驗證 (Constraint Check)**: 求解器輸入模式下，系統會即時分析行、列、宮格。一旦發現重複數字，該格子閃爍紅色警示（`.cell.conflict`），頂部顯示淡入紅色警告看板，並**自動禁用「求解」按鈕**以防崩潰。
*   **F10: 輸入與解答視覺區隔 (Color Coding)**: 求解成功後，使用者自行輸入的原始數字保持深藍粗體（`.cell.user-input`），而由演算法算出的解答數字呈現**翡翠綠（`.cell.solver-solved`）與淡綠背景**，一目了然。
*   **F11: 一鍵「全部清除」與狀態解鎖**: 工具列整合 [復原 | 清除 | 全部清除] 圓形按鈕欄。點擊「全部清除」後，不僅重設盤面為零，更會**徹底重設獲勝標記（`isGameWon = false`）與唯讀鎖**，使棋盤恢復自由點擊與編輯。

### 2.3 使用者身份驗證與角色權限
*   **F12: 磨砂玻璃驗證彈窗 (Glassmorphic Auth Modal)**: 提供極具質感的高斯模糊遮罩登入與註冊彈窗，支援非同步重複提交防護（處理中按鈕 Disabled）。
*   **F13: 身分識別徽章 (Role Badges)**: 登入成功後，右上角及手機側邊欄會顯示 `👤 [帳號] - [角色徽章]`，其角色徽章採用精心調配的 HSL 漸層配色：
    *   `standard` (普通玩家)：清新綠色漸層
    *   `power` (資深玩家)：尊爵藍色漸層，特權解鎖「介面主題切換」與「個人詳細統計數據（平均解題時間等）」。
    *   `admin` (管理員)：奢華金色漸層，特權解鎖「管理員後台（可檢視系統操作日誌與使用者狀態變更）」。
*   **F14: 登入狀態持久化 (Session Keep-Alive)**: 在頁面加載第一時間，透過 Fetch API 非同步向後端發送 Session 狀態檢查，確保玩家 F5 重新整理時不會被迫登出。

### 2.4 社群與系統安全監控
*   **F15: 多態好友系統 (Friends Network)**: 支援玩家間發送好友請求，具備等待接受（`pending`）、已接受（`accepted`）及黑名單（`blocked`）三種狀態。
*   **F16: 反作弊分析與安全審計日誌 (System Logs & Anti-Cheat)**:
    *   後端針對每次登入嘗試、異常提權或異常短用時通關等操作進行主動審計。
    *   若玩家發生密碼比對失敗、或在遊戲中觸發作弊邏輯，系統會自動在日誌寫入一筆 `is_suspicious = 1` 的警示標記，利於管理員於後台進行安全防護。

### 2.5 移動端與響應式優化 (iPhone 12 Pro RWD)
*   **F17: 手機側邊漢堡選單 (Mobile Sidebar)**: 電腦端右上角選單於手機版自動收納至左上角漢堡選單內，並動態整合個人登入徽章與控制按鈕。
*   **F18: 橫向滑動藥丸標籤列 (Mobile TOC Pills)**: 行動端下，規則彈窗（Rules Modal）原本死板的左側目錄重寫為**可橫向滑動的扁平化藥丸標籤列**。當前選取項變為高對比藍色，未選取項為優雅淡灰。
*   **F19: 微型滾輪指示器 (Micro Scrollbar Indicator)**: 在標籤滾動容器下方提供高度僅為 `4px` 的極細圓角淡灰滾動條，提供明確的手勢滾動視覺指引，且與標籤留有安全間距。
*   **F20: 規則星芒列表 (✦ Star Bullets)**: 規則面板正文去除了傳統粗糙的 `ul` 黑色圓點，全部改用柔和 HSL 主色調的 **星芒符號 `✦`** 並增加行高至 `1.7`，解決小螢幕的視覺壓迫感。

### 2.6 管理者安全主控台 2.0 (Admin Portal v2.0) [NEW]
*   **F21: 獨立全網頁管理主控台 (Standalone Portal)**: 徹底棄用舊版擁擠的 Modal 彈窗，將整個後台釋放為專業的 Sidebar + Glassmorphism 三欄式儀表板（`admin.html`），以全新獨立分頁開啟，保障管理空間與視覺美感。
*   **F22: 用戶帳號狀態控管 (User Status Controls)**: 提供對全系統用戶之啟用 (active)、停用 (suspended) 與封鎖 (blocked) 狀態設定。停用帳號將立刻強制踢除 Session 登出，且管理員操作時具備最高管理員自我降權與封鎖之安全防護機制。
*   **F23: 系統一鍵維護模式 (Maintenance Mode)**: 後端提供全域開關設定，一鍵啟用維護模式。啟用期間非管理員角色之玩家登入或呼叫 API 時會被安全攔截並回應 503 維護代碼，僅限最高管理員可通行調試。
*   **F24: 進階安全審計日誌篩選 (Advanced Filtering)**: 提供全頁式審計日誌清單，支援依使用者名稱、特定安全行為標記、時間範圍 (起/迄) 進行多條件交叉過濾，並可一鍵顯示 `is_suspicious = 1` 的反作弊異常安全日誌。

## 3. 非功能功能需求 (Non-functional Requirements)
*   **NF01: 全響應式介面 (RWD)**: 完美適應 375px (iPhone 12 Pro)、412px 至 1920px 寬度的所有螢幕。
*   **NF02: 後端安全保障**: 所有後端資料庫互動必須 100% 使用 PHP 8.4 `mysqli` 預處理陳述式 (Prepared Statements)，杜絕 SQL 注入。
*   **NF03: 零延遲體驗**: 客戶端矩陣求解演算法必須在 15ms 內輸出解答。
*   **NF04: 資料庫相容性**: 完全相容 MariaDB 10.11 規格，且支援在無伺服器 root 建立資料庫權限的限制下，於預配的個人專屬同名資料庫中正常運作。
