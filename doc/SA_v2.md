# 系統分析 v2 (SA_v2.md) - 純前端架構版

## 1. 簡化後的目錄結構
由於移除 Python 後端需求，專案目錄結構更為精簡：

```text
數獨/
├── frontend/               # 核心網頁目錄 (部署此資料夾即可)
│   ├── index.html          # 主入口
│   ├── style.css           # 視覺樣式
│   └── script.js           # 整合了後端邏輯與前端互動的 JS 檔案
└── doc/                    # 系統規格書目錄 (v1 與 v2 共存)
```

## 2. 單機運算架構分析
本版本達成「去中心化」，使用者下載網頁後即可在本地瀏覽器獨立運作。

### 2.1 邏輯整合
原本在 Python 運行的「矩陣運算」現已改為在 JavaScript 的 `SudokuGenerator` 模組執行。
*   **優勢**: 無需網路通訊、無伺服器延遲、部署難度降為零。
*   **效能**: 現代瀏覽器處理 9x9 矩陣的回溯法求解僅需不到 10ms，完全能滿足即時互動需求。

### 2.2 離線持久化
*   利用 `LocalStorage` 替代資料庫，實現進度與設定的永久保存。
*   玩家重啟瀏覽器後，`window.onload` 會優先檢查本地存檔。

## 3. 核心演算法流程 (JS 實作)
1.  **初始化**: 洗牌 (Shuffle) 1-9 序列，確保每次生成的數獨都不相同。
2.  **生成**: 遞迴填入數字，直到找到第一組合法完整盤面。
3.  **挖洞**: 隨機選擇格子歸零，直到達到目標提示數。
4.  **提示引導**: 實作於前端 `startTutorialHint()`，透過 JS 動態分析關聯區域。
