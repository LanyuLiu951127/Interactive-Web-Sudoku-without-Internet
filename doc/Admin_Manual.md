# 數獨全端平台：管理者與運維操作手冊 (Admin & Operations Manual)

## 📌 版本控制與單一事實來源 (SSOT)
*   **手冊版本**：v3.1
*   **系統架構**：Ubuntu 24.04 + Nginx 1.24 + PHP 8.4 + MariaDB 10.11
*   **主編專家**：BMAD 聯合專家小組 (@DMS、@BMM、@QAS、@OPS)

本手冊旨在引導平台管理員與運維人員（Ops）安全地進行管理身份驗證、操作數獨求解器、以及利用後端資料庫進行防作弊安全審計。

---

## 🔑 第一章：管理員身分驗證與登入流程 (Admin Login)

系統內置了管理員特權帳號，用以進入專屬的管理看板與執行進階審計。

### 1.1 管理員登入 SOP
1.  **開啟系統主頁**：使用瀏覽器造訪數獨平台首頁。
    > [📸 這裡請截取：數獨首頁加載完成，右上角顯示「註冊」與「登入」按鈕之畫面]
2.  **開啟登入彈窗**：點擊右上角（桌機端）的 **「登入」** 按鈕，或開啟手機左上角漢堡選單點選 **「登入」**。系統將彈出極具質感的高斯模糊磨砂玻璃彈窗。
3.  **輸入管理員憑證**：
    *   **管理員帳號**：`admin_liu`
    *   **預設密碼**：`123456`
    > [📸 這裡請截取：磨砂玻璃登入彈窗中已填妥 admin_liu 帳密，且「登入中...」按鈕防重複提交點按之畫面]
4.  **送出驗證**：點擊「登入」按鈕。系統將採用安全預處理陳述式比對 Bcrypt 雜湊值。
5.  **狀態保留與 Badge 驗證**：
    *   驗證成功後，彈窗自動關閉。
    *   右上角與手機側邊欄將即時渲染專屬的金色漸層徽章：**👤 admin_liu - 管理員**。
    *   按下 `F5` 重新整理，網頁會自動向後端呼叫 `check_session.php`，確保登入狀態維持不中斷。
    > [📸 這裡請截取：管理員成功登入，右上角顯示金色尊貴「管理員」HSL Badge 之畫面]

### 1.2 登入異常處理與安全防護
*   若輸入錯誤密碼，系統會顯示紅色提示訊息：「帳號或密碼錯誤」。
*   **防爆破監控**：後端 PHP 會立即在 `system_logs` 自動寫入一筆 `login_failed` 紀錄，並主動將 **`is_suspicious` 欄位標記為 `1`**（可疑操作），供後續運維人員追蹤暴力破解行為。

## 🧮 第二章：數獨求解器標準操作 SOP (Sudoku Solver)

數獨求解器供玩家與管理員輸入實體報紙或外部卡關題目，並由前端回溯演算法進行即時求解。

### 2.1 進入與輸入階段 (Input & Validation)
1.  **進入求解模式**：開啟左側選單，點擊 **「數獨求解器」**。
    *   系統將切換為求解器介面：隱藏計時器、錯誤次數，並將棋盤徹底清空。
    *   此時，1-9 數字鍵上方工具列會重組為求解器專屬的：`[ ↶ 復原 | 🗑️ 清除 | 🔄 全部清除 ]` 欄位。
    > [📸 這裡請截取：切換至數獨求解器模式後，上方工具列呈現復原、清除與全部清除三元按鈕之畫面]
2.  **手動輸入題目**：點選棋盤格子，輸入已知題目。
    *   手動輸入的數字呈現 **深藍色粗體**（`.cell.user-input`）。
3.  **即時約束驗證 (Constraint Check)**：
    *   若在同行、同列或同宮格填入重複的非法數字，發生衝突的格子會立刻呈現 **粉紅色呼吸背景與紅色粗體字**（`.cell.conflict`），發出呼吸閃爍特效。
    *   棋盤上方立刻淡入紅色警告看板：「相同的數字不能放在同一列、行或宮！」
    *   **「求解」按鈕將自動變為灰色並禁用 (disabled)**，徹底阻斷非法求解崩潰。
    > [📸 這裡請截取：求解器輸入衝突時，格子呈現紅色呼吸燈，上方警告看板顯示，且求解按鈕變灰之畫面]

### 2.2 求解與區隔階段 (Solving & Output)
1.  **執行求解**：在排除所有紅色衝突後，點擊最下方的綠色 **「求解」** 按鈕。
2.  **視覺精準區隔**：
    *   演算法將於 15ms 內算出解答。
    *   **使用者原始輸入**：依然維持原汁原味的 **深藍色粗體**，便於比對。
    *   **系統自動解答數字**：套用 `.cell.solver-solved` 樣式，呈現優雅的 **翡翠綠字體與薄荷淡綠底色**，高低分明。
    > [📸 這裡請截取：求解完成後，深藍色原始數與翡翠綠自動解答數在棋盤上完美區隔之畫面]

### 2.3 清空與解鎖階段 (Reset & Unlock)
1.  **一鍵清除**：點選工具列上的 **「🔄 全部清除」** 按鈕。
2.  **狀態徹底解鎖**：
    *   棋盤的所有數值歸零，所有的衝突紅字與頂部警告看板一瞬間消失。
    *   **死鎖完全釋放**：系統會自動重置全域狀態 `isGameWon = false`，所有棋盤格子恢復 100% 可自由點選、編輯與輸入的狀態。
    > [📸 這裡請截取：點擊全部清除後，棋盤完璧歸趙、完全清空且各格子可重新自由點按輸入之畫面]

## 🔒 第三章：後端審計運維與防作弊分析 (Database Audit)

後端透過 `system_logs` 表，自動對整個數獨平台進行安全監控。運維人員（Ops）必須定期登入 phpMyAdmin 進行異常排除。

### 3.1 登入 phpMyAdmin 查詢異常日誌
1.  **登入 phpMyAdmin**：造訪學校主機或本地 `http://localhost/phpmyadmin`。
2.  **選擇資料庫**：在左側導覽列點選預配的個人資料庫 `st111534105`（或平台資料庫）。
    > [📸 這裡請截取：phpMyAdmin 資料庫介面，左側選中專屬同名資料庫之畫面]
3.  **查詢防作弊日誌**：
    點擊上方 **「SQL」** 分頁，輸入並執行以下安全審計查詢指令：
    ```sql
    SELECT * FROM `system_logs` WHERE `is_suspicious` = 1 ORDER BY `created_at` DESC;
    ```
    > [📸 這裡請截取：在 phpMyAdmin 中輸入查詢 is_suspicious = 1 SQL 指令之畫面]
4.  **分析審計欄位**：
    *   `is_suspicious` 欄位為 `1` 的紀錄代表系統偵測到異常操作。
    *   **典型作弊與異常場景說明**：
        1.  `action = 'login_failed'`：IP 在短時間內產生大量失敗紀錄，表示遭受到暴力破解攻擊。
        2.  `action = 'cheat_attempt'`：通關用時小於合理常規秒數（如 5 秒內完成困難難度），被系統反作弊模組自動標記。
        3.  `action = 'privilege_escalation'`：非管理員 Session 嘗試調用敏感的管理員 API。
    > [📸 這裡請截取：phpMyAdmin 執行結果顯示 is_suspicious = 1，包含連線 IP 與作弊原因之日誌畫面]

### 3.2 FileZilla 運維安全部署守則
當進行系統日常更新與部署時，運維人員必須遵守以下規範：
1.  **強制 UTF-8 傳輸**：在 FileZilla 中強制將字元集設為 UTF-8，防止後端 `db_connect.php` 發生亂碼。
2.  **敏感檔案隔離**：嚴禁將 `init_db.sql`、`doc/` 規格書或本地 `.git` 資料夾部署至公網 `/public_html/` 之下，防止伺服器敏感資訊外洩。
